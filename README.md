# Send_to

Windowsの「送る」メニューに登録して使用する3つのツールをまとめたリポジトリ。いずれもファイルを右クリック→「送る」で起動し、手作業による定型処理を自動化する。

---

## 収録ツール一覧

| ファイル名 | 対象ファイル | 主な処理 |
|-----------|------------|---------|
| ファイル結合_CSV.py | MIRAIMILエクスポートCSV（複数） | CSV結合 + 種別に応じたSQLite DB一括生成 |
| 受渡場所ファイル_コピー.py | 受渡場所マスタCSV | 所定フォルダへの配置・列加工 |
| xlsx_merge.py | Excelファイル（複数） | xlsx結合・列数制限・元ファイル削除 |

---

## ツール詳細

### 1. ファイル結合_CSV.py — CSV結合とDB一括生成

MIRAIMILから分割エクスポートされた同種CSVを右クリック→「送る」で複数選択して渡すと、結合・ファイル種別判定・DB生成を一括実行する。

起動時に OneDrive 共有フォルダの `業務用pythonアプリ最新版/ファイル結合_CSVアプリ/update` を確認し、`version.txt` の先頭行が現行バージョンより新しい場合は同名スクリプトで自動更新する。

ファイル種別（受注・発注・入荷・出荷・仕入）はダイアログで選択し、選択内容に応じて以下の後処理が連鎖する。

| ファイル種別 | 出力先 | 生成されるDB・ファイル |
|------------|--------|----------------------|
| 【標準】_受注 | 受注残フォルダ | Customer_List.dbから社員情報を付与 / 受注案件状況確認.db / order_data.db |
| 【標準】_発注 | 発注残フォルダ | Outstanding orders.csv（列マッピング変換済み）/ orders_info.db / Outstanding_orders.db / purchase_order_data.db |
| 【標準】_入荷 | 入荷実績フォルダ | arrival_data.db |
| 【標準】_出荷 | 出荷実績フォルダ | shipped.db（重複UNIQUE制約付き） |
| 【標準】_仕入 | 仕入実績フォルダ | Purchase_price.db（赤伝相殺処理付き） |

処理完了後、結合元のCSVファイルは自動削除される（定常ファイル名のものはスキップ）。

CSV結合と同時にDBを生成する理由は2つある。第一に、このツールで生成したDBを入出荷状況確認アプリ・各種データ処理ツール等の下流ツールが参照しているため、CSVを結合した時点でDBも最新状態に保つ必要がある。第二に、CSVのままでは検索・結合処理が遅く、SQLiteに変換することで参照パフォーマンスが向上する。「CSVを所定フォルダに置く」という1アクションで、下流のすべてのツールが使えるデータ状態になる設計にした。

### 2. 受渡場所ファイル_コピー.py — 受渡場所マスタの配置と加工

MIRAIMILからエクスポートした受渡場所マスタCSVを右クリック→「送る」で渡すと、以下を一括実行する。

- 所定フォルダ（受渡場所一覧フォルダ）へ移動し `受渡場所.csv` にリネーム
- 既存ファイルが存在する場合は `受渡場所_backup.csv` としてバックアップを作成してから上書き
- 「得意先コード」と「得意先枝番」を結合した列（`得意先コード-得意先枝番`）を先頭に追加
- 6列目の値を2桁ゼロパディングに変換

マスタの更新が「ファイルを渡すだけ」で完結し、手動でのフォルダ移動・リネーム・列加工が不要になる。

### 3.ファイル結合_xlsx.py — Excelファイルの結合

2つ以上のxlsxファイルを右クリック→「送る」で渡すと、以下を実行する。

- 1つ目のファイルをヘッダー付きで読み込み、2つ目以降はヘッダー行をスキップして結合
- 結合後にV列まで（先頭22列）のみを出力
- タイムスタンプ付きファイル名（`merged_YYYYMMDD_HHMMSS.xlsx`）で元ファイルと同フォルダに保存
- 結合元ファイルを自動削除

## アーキテクチャ上の工夫

### 「送る」を起動インターフェースとして統一する

3ツールはいずれも起動方法を「右クリック→送る」に統一した。GUIランチャーを別途開く必要がなく、対象ファイルを選んだその場で処理が始まる。操作手順を最小化することで、定型作業における「ツールを探して起動する」というオーバーヘッドをなくした。

### ファイル結合_CSV.py：CSVとDBを常に同期させる設計

CSV結合をトリガーとしてDB生成まで連鎖させることで、「CSVは更新したがDBは古いまま」という状態が発生しない設計にした。下流ツールはDBを参照する前提で作られており、このツールが「データ更新の窓口」として機能している。

### ファイル結合_CSV.py：赤伝相殺処理を2パスで実装する

仕入単価DB（Purchase_price.db）の生成では、赤伝（返品・訂正）レコードとその相殺対象を両方除外する必要がある。1回のスキャンでは相殺対象を特定できないため、第1パスで赤伝フラグが立つレコードの組み合わせを収集し、第2パスで当該組み合わせを除外してDBに挿入する2パス方式を採用した。

### 受渡場所ファイル_コピー.py：バックアップを自動作成してから上書きする

マスタファイルの上書き前に自動でバックアップを作成する設計にした。誤ったファイルを渡してしまった場合でも、直前のマスタ状態に手動で戻せる。

## セットアップ

各スクリプトのショートカットをWindowsの「送る」フォルダに配置する。
```
shell:sendto
（エクスプローラーのアドレスバーに入力してフォルダを開く）
```

配置後、対象ファイルを右クリック→「送る」に各ツール名が表示される。

## 注意事項

- 出荷依頼書ファイル整形.pyおよび発注ファイル_必要情報コピー.pyは旧プログラムのため、削除予定
- ファイル結合_CSV.py：OneDriveの同期完了後に実行すること。同期前に実行すると古いCSVやDBが参照される
- ファイル結合_CSV.py：入力CSVはMIRAIMILからのエクスポートファイル（cp932）であること
- ファイル結合_CSV.py：結合元CSVは処理完了後に自動削除される。定常ファイル名（【標準】_受注.csv等）のファイルは削除対象外
- ファイル結合_CSV.py：自動アップデート用 `update` フォルダには `version.txt`（1行目にバージョン）と `ファイル結合_CSV.py`（最新本体）を配置すること
- xlsx_merge.py：結合後はV列（先頭22列）のみが出力される。列数が異なるファイルを渡す場合は事前に確認すること
- 社内の受注・発注・仕入・顧客情報が含まれるため、社外持ち出し・公開リポジトリへのアップロードは不可
